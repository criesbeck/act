The template format for a user story is "A [type of user] can [interact with the product] in order to [achieve some goal]."
The user story is the basic unit of work and progress in agile. A user story is not about what code gets written, but what goal a user will be able to accomplish.
A user story is not a requirement. It is just a token for future conversation about the real requirement.
A user story is not a scenario. A user story is short, typically one sentence. A user story is focused on one goal. A user story is defined for a role, not a persona.
Ironically, most user stories should not use the word "user". "User" is too vague and ambiguous. Most products are used by people with specific roles, e.g., players, buyers, sellers, authors, editors, and so on. Different roles have different needs.
A commonly recommended user story template is "As a type of user, I can do something in order to achieve some goal." The goal makes all the difference. "As a buyer, I can sort a list of products." Sort how? By price? By rating? By popularity? It all depends on "why".
